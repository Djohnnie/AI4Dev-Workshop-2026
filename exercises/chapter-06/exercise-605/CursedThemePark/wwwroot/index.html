Cursed Theme Park Checkout

A haunted .NET checkout flow for debugging practice. Trigger the cursed scenarios below, inspect the crash, and use Copilot to trace which assumption fails first.

How participants can work this exercise

  1. Run one cursed scenario from the UI or CursedThemePark.http.
  2. Inspect the failing request, logs, and stack trace.
  3. Narrow the fault to one broken assumption.
  4. Fix it and turn the repro into a regression test.
